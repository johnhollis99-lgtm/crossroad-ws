# Region narration: soul-doctrine content required (deviates from addendum §3.5 length cap)

**Created:** 2026-05-15
**Adopted:** 2026-05-18 (curator-locked during Track 1 narrator_a_family tone-check)
**Phase:** E2 (region narration pre-generation)
**Status:** Adopted; supersedes the strict 60–90 sec / 150–200 word cap in addendum §3.5 for cases where soul-doctrine layers all need to land.

## Decision

Region narration prompt templates carry a hard requirement that when source material supports them, **all three soul layers — geology, geography, anthropology — must appear in the narration**. Indigenous-peoples mention specifically must use present-tense, living, named framing. To accommodate this requirement when the source supports it, the addendum §3.5 length cap of 60–90 seconds (150–200 words) stretches up to ~120 seconds (~280 words). The shorter target remains the default; the longer cap is invoked only when content density requires it. A narration that hits the 90-second mark by dropping a soul layer the source supports is *incomplete*, not *concise*.

Existing tightenings from the 2026-05-15 narrator_a × family tone-check round remain in effect: no listener-as-protagonist framing ("imagine you are…"); pointing OK ("the Sierra rises to the west") but puppeteering ("look west toward the granite wall") not OK; strict present-tense for indigenous peoples ("the Paiute remain here", not "have inhabited for thousands of years" — the latter is present-perfect and slides toward past-tense in audio delivery).

## Reasoning

The addendum §1 Soul Doctrine is unambiguous: *"the soul of the app is geology, geography, and anthropology."* If a region narration omits the indigenous layer when the source supports it, the app's stated identity isn't on the page. The 60–90 sec cap in §3.5 was a runtime-friendliness target written before the soul doctrine's full implications for region content were stress-tested via real renders. The Mono Basin v1 narration (rendered 2026-05-15) hit 197 words but dropped the Paiute presence into a single past-tense-ish sentence at the end; the Mono Basin v2 re-render (rendered 2026-05-18 via fresh Haiku draft on the same template) dropped the indigenous layer entirely. Both were within the length cap; both were soul-incomplete by the doctrine's standards.

Running long (110–280 words past the 90-second target, ~30 sec extra audio) is acceptable when the source supports the depth. Most regions will land in the 150–200 default; the soul-dense ones (Owens Valley, Mono Basin, regions with named volcanic histories or active indigenous presences) take the extra time the doctrine demands. The trade-off — slightly longer runtime per region boundary cross during drive — is the price of staying truthful to the app's stated identity. Reversible later if usage data shows the longer renders skew skip rates; the template's length-cap envelope is one prompt edit away.
